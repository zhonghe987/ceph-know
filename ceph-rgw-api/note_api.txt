1,创建用户
key-type必须为s3


2，同个用户创建存在的桶名依然为200

3，修改用户信息中，指定秘钥同时也指定access_key，指定access_key也要指定秘钥，该用户增加一对access_key和sec_key，不影响之前的存在的秘钥对。

4,桶名必须唯一，表现为不同的用户不能创建相同的桶名，同个用户创建创建两个相同的桶名可以，后者不覆盖前者的数据，但实际上只有一个桶。

5,s3 api 创建文件夹需要添加'/'，例如 wewe/

7,设置为公共读写权限的桶，改变所者有对其为读或者写权限后，其他用户对该桶丧失读写的权限

8，api创建子用户，默认是创建swift类型。指定key-type类型为s3时，radosgw-admin需指定access-key就可以创建子用户，api不支持s3类型子用户创建。

9，子用户重新设置桶的权限为失败报403。

10,修改用户对公共读写桶的权限为读或写或full，其他用户则失去对该桶的读写权限
